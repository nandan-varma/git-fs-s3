```typescript
import { S3Client } from '@aws-sdk/client-s3';
import { S3ObjectStore } from 'git-fs-s3/s3';

const store = new S3ObjectStore({
  client: new S3Client({
    region: 'auto',
    endpoint: process.env.R2_ENDPOINT, // omit for AWS S3
    credentials: {
      accessKeyId: process.env.R2_ACCESS_KEY_ID!,
      secretAccessKey: process.env.R2_SECRET_ACCESS_KEY!,
    },
  }),
  bucket: 'my-git-repos',
  contentType: (key) =>
    /\/(HEAD|config)$|\/refs\//.test(key) ? 'text/plain' : undefined,
});
```

<Aside type="tip">
	One instance per bucket/prefix — but reuse a single `S3Client` across
	every store you create. A busy repo page can mean hundreds of object
	reads; a shared client pools HTTP connections instead of reconnecting
	per store.
</Aside>

`@aws-sdk/client-s3` is an optional peer dependency, loaded only when you
import this subpath — the root package and `/http`/`/ops` never pull it in.
